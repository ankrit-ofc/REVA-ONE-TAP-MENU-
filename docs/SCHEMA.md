# Database Schema Reference

> ⚠️ **STALE — do not trust for current schema.** This file reflects the
> Phase 1 baseline only and is behind migrations `0004`–`0018` (food types,
> geofencing, printer settings, AR capture, category nesting, KOT print jobs,
> waiter calls, device tokens, and more). The migrations in
> `backend/alembic/versions/` are the source of truth until this file is
> regenerated.

Generated by Phase 1. All monetary values are `NUMERIC` — never `FLOAT`.
All PKs are `UUID`. All timestamps are `TIMESTAMPTZ` (UTC).
Tenant tables carry `restaurant_id UUID NOT NULL REFERENCES restaurants(id)`.

## Row-Level Security

All 15 tenant tables have RLS enabled with a single policy:

```sql
CREATE POLICY tenant_isolation ON <table>
USING (restaurant_id = current_setting('app.current_restaurant_id', TRUE)::uuid);
```

Phase 2 sets `app.current_restaurant_id` per request from the verified JWT/session.
App-layer `WHERE restaurant_id = :id` filtering remains mandatory; RLS is the net.
